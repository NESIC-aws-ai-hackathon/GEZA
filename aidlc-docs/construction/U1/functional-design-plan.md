## 質問事項

### Q1: アバターの初期表示仕様

トップ画面に表示する謝罪ボスのアバターは、ユーザーがログイン前から表示しますか？

A) ログイン前でも固定アバター（デフォルト設定）を表示する  
B) ログインしていない場合はアバターなし、ログイン後に表示  
C) ログイン前はアニメーションのみ（アバターは後から生成する U2 まで不要）

[Answer]: A 固定ではなくランダムアバターにしよう

---

### Q2: ログイン画面とトップ画面の関係

index.html（ログインページ）と トップ画面（モード選択・アバター表示）の構成はどちらですか？

A) 1ページ構成: index.html にログインフォームとトップ画面を両方持ち、認証状態でセクションを切り替える  
B) 2ページ構成: index.html はログイン専用、認証後に top.html（またはindex.htmlのtopセクション）に遷移する  
C) SPA: index.html 1本でハッシュルーティング（#login / #top / #mode-select）で切り替える

[Answer]: A

---

### Q3: Cognito サインアップフロー

AC-1「サインアップできる」の実装範囲として、U1 ではどこまで対応しますか？

A) メール確認コード入力まで（フルサインアップフロー）  
B) サインアップフォーム送信のみ（確認コード入力は別画面・U1 スコープ外）  
C) 管理者作成のみ（AGENTS.md の方針通り `AllowAdminCreateUserOnly=true`、サインアップ UI は不要）

[Answer]: C

### Q4: MFA（TOTP）の UI 対応

Cognito User Pool は `MfaConfiguration: ON` ですが、TOTP のセットアップ/入力 UI を U1 で実装しますか？

A) U1 でフルMFA対応（TOTPセットアップ + コード入力 UI 実装）  
B) U1 はメール/パスワードのみ。MFA UIは将来対応  
C) 管理者のみ作成するため、エンドユーザー向けMFA UI は不要（管理者が AWS Console でユーザー作成・MFA登録）

[Answer]: C

### Q5: モード選択の表示対象

AC-4「モード選択導線」に4つのモードが挙げられていますが、U1 のスコープとして表示するモードを確認します。

A) 4モード全て（実案件・ストーリー・カルテ・上司向け）を表示（未実装モードはグレーアウト）  
B) 実装済みモードのみ表示（U1 完了時点では実案件モードのみ）  
C) 実案件 + ストーリーの2モードのみ表示（カルテ・上司向けは後で追加）

[Answer]: A

### Q6: 3ゲージ（怒り度・信頼度・難易度）の初期値

AC-3「ステータスゲージが表示される」について、謝罪ボス未設定のトップ画面での初期値はどうしますか？

A) 固定値（例: 怒り度 MAX・信頼度ゼロ・難易度中）をデフォルトとして常に表示  
B) ゲージはU2（相手生成後）まで表示しない  
C) 演出として全ゲージが揺らいで「?」表示（相手未設定を示す）

### Q7: 認証後のセッション管理

U0 の `auth.js` では `sessionStorage` を使っていますが、AC-3「ブラウザを閉じて再度アプリを開いても再ログイン不要」に対応するには `localStorage` が必要です。どちらを使いますか？

A) localStorage を使用（ブラウザ再起動後も認証維持）  
B) sessionStorage を使用（タブ/ウィンドウを閉じたら再ログイン必要）  
C) Cognito SDK のデフォルト動作に任せる（SDK が適切に管理）

[Answer]: C XSS 対策 Cognito refreshToken のみ localStorage + idToken/accessToken は メモリ
