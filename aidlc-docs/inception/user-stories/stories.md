## 感情定義（30種類）

ストーリー内で参照される謝罪ボスの感情ラベル一覧。facesjs SVG + CSS表情制御で表現する。

| # | 感情ID | 日本語名 | カテゴリ | 表現（概要） |
|---|--------|---------|---------|------------|
| 1 | rage | 激怒 | 強い怒り | 眉を大きく吊り上げ、目を見開き、口を大きく開く。画面揺れエフェクト |
| 2 | anger | 怒り | 怒り | 眉を吊り上げ、目を細め、への字口 |

| 6 | disappointment | 失望 | 悲しみ系 | 眉を下げ、目を細め、口角を下げる |
| 7 | sadness | 悲しみ | 悲しみ系 | 眉を八の字、目を潤ませ、口を小さく開く |
| 8 | contempt | 軽蔑 | 冷たい系 | 片眉を上げ、口角を片側だけ上げる |

| 14 | impatience | 焦燥 | 不満 | 眉を寄せ、目を左右に動かす、足踏みモーション |
| 15 | weariness | 疲弊 | 諦め系 | 目を半開き、眉を下げ、ため息 |
| 16 | resignation | 諦め | 諦め系 | 目を閉じ、首を横に振る |
| 17 | coldness | 冷淡 | 冷たい系 | 無表情、目が据わる、口一文字 |
| 18 | intimidation | 威圧 | 強い怒り | 顎を上げ、目を見下ろす、じわじわ近づくモーション |
| 19 | sarcasm | 皮肉 | 冷たい系 | 口角を片方だけ大きく上げ、首を傾げる |
| 20 | bitterness | 苦々しさ | 悲しみ系 | 唇を噛む、眉を寄せ、顔をそむける |
| 21 | confusion | 困惑 | 中立 | 眉をハの字、口を半開き、首を傾げる |
| 22 | hesitation | 戸惑い | 中立 | 視線を逸らす、口を開閉する |
| 23 | thinking | 思案 | 中立 | 目を上に向け、口を閉じ、腕を組むモーション |

| 25 | empathy | 共感 | 好転 | 眉を寄せつつも目を柔らかく、小さくうなずく |
| 26 | relief | 安堵 | 肯定 | 目を閉じてゆっくり開く、肩の力が抜けるモーション |
| 27 | acceptance | 納得 | 肯定 | 眉を自然に、目を穏やかに、口角をわずかに上げる |
| 28 | appreciation | 感謝 | 肯定 | 目を潤ませ、深くうなずく、口角を上げる |
| 29 | satisfaction | 満足 | 肯定 | 目を細めて微笑む、大きくうなずく |

**エフェクト**: rage/shock/forgiveness 等の特に強い感情では画面揺れ・画面明暗変化のエフェクトを追加する。

# Journey 1: アプリを知る（トップ画面）

## US-101: トップ画面で謝罪ボスに出会う

| 項目 | 内容 |
|------|------|
| **ペルソナ** | Kenta / Misaki / Seiichi（全員） |
| **優先度** | P0（必須） |
| **Story Points** | 5 |
| **依存** | なし（最初に実装） |
| **Epic** | Epic 1 |

**ユーザーとして**、アプリを開いた瞬間に謝罪ボスのアバターが表示されてほしい。  
**なぜなら**、「謝罪対象と向き合うサービス」だと直感的に理解したいから。

### 受け入れ条件

**AC-1: トップ画面にアバターが表示される**
- **Given** ユーザーがアプリのURLにアクセスしたとき
- **When** トップ画面が読み込まれる
- **Then** facesjs SVGアバター（謝罪ボス）が画面上部中央に表示され、アイドルアニメーション（瞬き・視線移動・頭の揺れ）が再生される

**AC-2: キャッチコピーが表示される**
- **Given** トップ画面が表示されたとき
- **When** ユーザーが画面を見る
- **Then** 「謝る前に、怒られておけ。」がアバター付近に表示される

**AC-3: ステータスゲージが表示される**
- **Given** トップ画面にアバターが表示されたとき
- **When** ユーザーがアバターを確認する
- **Then** 怒り度・信頼度・難易度の3ゲージが視覚的に表示される

**AC-4: モード選択導線がある**
- **Given** トップ画面が表示されたとき
- **When** ユーザーがモード選択エリアを見る
- **Then** 「実案件モード」「ストーリーモード」「謝罪カルテ」「上司向けフィードバック」への導線ボタンが表示される

**AC-5: スマホ表示に対応する**
- **Given** ユーザーがスマホ（375px幅）またはPCからアクセスしたとき
- **When** 画面が表示される
- **Then** スマホ幅375pxでレイアウトが崩れず自然に表示される。PCではスマホ幅で中央表示される

## US-102: Cognito認証でログインする

| 項目 | 内容 |
|------|------|
| **ペルソナ** | Kenta / Misaki / Seiichi（全員） |
| **優先度** | P0（必須） |
| **Story Points** | 8 |
| **依存** | なし（インフラと同時実装） |
| **Epic** | Epic 1 |

**ユーザーとして**、アカウントを作成してログインしたい。  
**なぜなら**、謝罪カルテや履歴を個人に紐づけて保存したいから。

### 受け入れ条件

**AC-1: サインアップができる**
- **Given** 未登録のユーザーがアプリにアクセスしたとき
- **When** サインアップフォームにメールアドレスとパスワードを入力して送信する
- **Then** Cognito User Pool にユーザーが作成され、確認メールが送信される

**AC-2: ログインができる**
- **Given** 登録済みのユーザーがアプリにアクセスしたとき
- **When** メールアドレスとパスワードを入力してログインする
- **Then** Cognitoトークンが取得され、認証済み状態でトップ画面に遷移する

**AC-3: 認証状態が維持される**
- **Given** ユーザーがログイン済みの状態で
- **When** ブラウザを閉じて再度アプリを開く
- **Then** トークンが有効な間は再ログイン不要でトップ画面が表示される

**AC-4: 未認証時にアクセスが制限される**
- **Given** 未ログインのユーザーが
- **When** 謝罪練習やカルテ画面にアクセスしようとする
- **Then** ログイン画面にリダイレクトされる

# Journey 2: やらかしを入力する（実案件モード）

## US-201: やらかし内容を自由入力する

| 項目 | 内容 |
|------|------|
| **ペルソナ** | Kenta / Seiichi |
| **優先度** | P0（必須） |
| **Story Points** | 5 |
| **依存** | US-103 |
| **Epic** | Epic 2 |

**ユーザーとして**、何をやらかしたかを自然文で入力したい。  
**なぜなら**、フォーム項目に分解するより、そのまま状況を説明したいから。

### 受け入れ条件

**AC-1: やらかし内容を自由テキストで入力できる**
- **Given** 実案件モードのやらかし入力画面を開いたとき
- **When** テキストエリアにやらかし内容を入力する
- **Then** 自由テキストが入力でき、文字数カウントが表示される

**AC-2: 補足情報を入力できる**
- **Given** やらかし内容を入力した状態で
- **When** 謝罪相手、関係性、謝罪期限、影響範囲の入力欄を確認する
- **Then** 各項目を任意で入力できる（未入力でもAIが推測して補完する）

**AC-3: AIが入力内容を要約する**
- **Given** ユーザーがやらかし内容と補足情報を入力して送信したとき
- **When** Bedrock APIが呼び出される
- **Then** 入力内容が謝罪案件として構造化・要約され、カテゴリ（顧客謝罪/上司謝罪/友人謝罪等）が推定表示される

**AC-4: AIがフォローアップ質問で補完する**
- **Given** 入力内容に不足がある場合
- **When** AIが分析を完了する
- **Then** 「影響を受けた人数は？」「過去に同様のミスは？」等のフォローアップ質問が表示され、ユーザーが追加情報を入力できる

## US-202: 謝罪相手をAIが生成する

| 項目 | 内容 |
|------|------|
| **ペルソナ** | Kenta / Seiichi |
| **優先度** | P0（必須） |
| **Story Points** | 8 |
| **依存** | US-201 |
| **Epic** | Epic 2 |

**ユーザーとして**、謝罪相手の怒り方や性格をAIに生成してほしい。  
**なぜなら**、相手が何に怒っているかを事前に理解して準備したいから。

### 受け入れ条件

**AC-1: 謝罪相手プロフィールが生成される**
- **Given** やらかし入力が完了した状態で
- **When** 「謝罪相手を生成」ボタンをタップする
- **Then** AIが以下を生成し表示する：
  - 相手タイプ（性格分類）
  - 怒り度・信頼度・許容度（初期値）
  - 怒りポイント（3件以上）
  - NGワード（3件以上）
  - 想定される第一声

**AC-2: アバターが生成される**
- **Given** 謝罪相手プロフィールが生成されたとき
- **When** アバター表示エリアを確認する
- **Then** 入力内容に応じたfacesjs SVGアバターが生成され表示される（seed値がプロフィールに紐づき、同じ相手なら同じ顔になる）

**AC-3: 架空の練習用であることが表示される**
- **Given** 謝罪相手が生成されたとき
- **When** プロフィール画面を確認する
- **Then** 「※実在の人物とは関係ありません。練習用の架空キャラクターです」と表示される

**AC-4: 謝罪練習に進める**
- **Given** 謝罪相手の確認が完了したとき
- **When** 「謝罪練習を始める」ボタンをタップする
- **Then** 生成された相手プロフィール・アバターを引き継いで、謝罪練習画面（Journey 4）に遷移する

# Journey 3: ストーリーに挑戦する

## US-301: 謝罪ストーリーを選択・生成する

|------|------|
| **ペルソナ** | Misaki（主） / Kenta / Seiichi |

**ユーザーとして**、毎回違う謝罪ストーリーに挑戦したい。  
**なぜなら**、ゲーム感覚で謝罪力を鍛えたいから。

### 受け入れ条件

**AC-1: 難易度・テーマを選択できる**

- **When** 難易度（初級/中級/上級/鬼級）とテーマ（顧客謝罪/上司謝罪/クレーム対応等）を選択する
- **Then** 選択が保存され、ストーリー生成に反映される

- **Given** 難易度とテーマが選択された状態で
- **When** 「ストーリー生成」ボタンをタップする
- **Then** AIがステージ名・背景ストーリー・謝罪対象プロフィール・NGワード・クリア条件・失敗条件を生成する

**AC-3: 謝罪ボスのステータスが表示される**
- **Given** ストーリーが生成されたとき
- **When** ボスプロフィール画面を確認する
- **Then** ボス名・難易度・怒り度・信頼度・許容度・クリア条件がソシャゲ風に表示される

**AC-4: アバターが生成される**
- **Given** 謝罪ボスプロフィールが生成されたとき
- **When** アバター表示を確認する
- **Then** ボスのプロフィールに紐づいたfacesjs SVGアバターが表示される（seedでボスごとに固定）

**AC-5: 固定ステージが用意されている**
- **Given** ストーリーモード画面を開いたとき
- **When** ステージ一覧を確認する
- **Then** 「データセンターに落ちた一滴」等のプリセットステージが選択可能である

# Journey 4: 謝罪を練習する（コア）

## US-401: 謝罪ボスがアバターとして画面に登場する

| 項目 | 内容 |
|------|------|
| **ペルソナ** | Kenta / Misaki / Seiichi（全員） |
| **優先度** | P0（必須） |
| **Story Points** | 8 |
| **依存** | US-202 または US-301 |
| **Epic** | Epic 4 |

**ユーザーとして**、謝罪対象が画面上で動いて反応してほしい。  
**なぜなら**、テキストだけよりも本当に相手に謝っている感覚を得たいから。

### 受け入れ条件

**AC-1: アバターが画面中央に表示される**
- **Given** 謝罪練習画面を開いたとき
- **When** アバターが読み込まれる
- **Then** facesjs SVGアバターが画面上部中央に表示され、30種類の感情表現に対応するCSS transform制御が可能な状態になる

**AC-2: アイドルアニメーションが再生される**
- **Given** アバターが表示された状態で
- **When** ユーザーが入力する前
- **Then** 瞬き（2.5〜6.5秒ランダム間隔）・視線移動（3.5〜8.5秒ランダム間隔）・頭の揺れ（headIdle CSS animation）が再生される

**AC-3: AI判定に応じて表情が切り替わる**
- **Given** ユーザーが謝罪を入力し、AIが感情ラベル（30種類中の1つ）を返却したとき
- **When** フロントエンドが感情ラベルを受信する
- **Then** アバターの眉・目・口がCSS transitionで滑らかに切り替わり、該当感情の表情が表示される（切り替え 0.5秒以内）

**AC-4: 強い感情で画面エフェクトが発生する**
- **Given** AIが rage / shock / forgiveness 等の強い感情を返却したとき
- **When** 感情が切り替わる
- **Then** 画面揺れ（rage/shock）または画面明暗変化（forgiveness）のエフェクトが再生される

**AC-5: 怒り度・信頼度ゲージが表示される**
- **Given** 謝罪練習画面が表示されたとき
- **When** ゲージエリアを確認する
