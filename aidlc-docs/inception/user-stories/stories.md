## プロダクトコンセプト

GEZAは**謝罪丸投げコンシェルジュ**です。  
「やらかした」と一言入れるだけで、謝罪角度判定→台本フル生成→タイミング・手土産提案まで全部AIがやります。あなたは頭を下げるだけ。

```
コア機能（入力一つで全自動）
    謝罪角度アセスメント  ← やらかしの深刻度をAIが0〜180°で即判定・ピクトグラム+スタンプ+SE音で演出
  謝罪台本フル生成      ← 相手分析 → NGワード → 第一声 → 全セリフ台本
  タイミング・手土産    ← いつ・どこで・何を持っていくかまで指示
  準備サポート          ← チェックリスト・フォローメール・再発防止策

オプション（やりたい人だけ）
  リハーサルモード      ← AI製台本の読み合わせ。失敗してもAIが書き直す
  上司向け指導練習      ← 独立フロー
```

**キャッチコピー：「土下座はあなたに、誠意はAIに。」**

## US-101: トップ画面で謝罪ボスに出会う

|------|------|
| **ペルソナ** | Kenta / Misaki / Seiichi（全員） |
| **優先度** | P0（必須） |
| **Story Points** | 5 |
| **依存** | なし（最初に実装） |
| **Epic** | Epic 1 |

**ユーザーとして**、アプリを開いた瞬間に謝罪ボスのアバターが表示されてほしい。  
**なぜなら**、「謝罪対象と向き合うサービス」だと直感的に理解したいから。

### 受け入れ条件

**AC-1: トップ画面にアバターが表示される**
- **Given** ユーザーがアプリのURLにアクセスしたとき
- **When** トップ画面が読み込まれる
- **Then** facesjs SVGアバター（謝罪ボス）が画面上部中央に表示され、アイドルアニメーション（瞬き・視線移動・頭の揺れ）が再生される

**AC-2: キャッチコピーが表示される**
- **Given** トップ画面が表示されたとき
- **When** ユーザーが画面を見る
- **Then** 「土下座はあなたに、誠意はAIに。」がアバター付近に表示される

**AC-3: ステータスゲージが表示される**
- **Given** トップ画面にアバターが表示されたとき
- **When** ユーザーがアバターを確認する
- **Then** 怒り度・信頼度・難易度の3ゲージが視覚的に表示される

**AC-4: モード選択導線がある**
- **Given** トップ画面が表示されたとき
- **When** ユーザーがモード選択エリアを見る
- **Then** 「実案件モード」「ストーリーモード」「謝罪カルテ」「上司向けフィードバック」への導線ボタンが表示される

**AC-5: スマホ表示に対応する**
- **Given** ユーザーがスマホ（375px幅）またはPCからアクセスしたとき
- **When** 画面が表示される
- **Then** スマホ幅375pxでレイアウトが崩れず自然に表示される。PCではスマホ幅で中央表示される

## US-207: 謝罪の角度をAIがアセスメントする

| 項目 | 内容 |
|------|------|
| **ペルソナ** | Kenta / Seiichi |
| **優先度** | P0（必須） |
| **Story Points** | 8 |
| **依存** | US-201 |
| **Epic** | Epic 2 |

**ユーザーとして**、やらかしの深刻度をAIが「謝罪の角度」として数値とピクトグラム・スタンプ・効果音で教えてほしい。  
**なぜなら**、自分が甘く見ていた問題をAIが客観的に評価することで、適切な謝罪姿勢を知りたいから。

### 受け入れ条件

**AC-1: AIが謝罪の角度（0〜180°）を算出する**
- **Given** やらかし内容が入力・分析された状態で
- **When** 「謝罪の角度を診断」ボタンをタップする
- **Then** Bedrock Nova Liteがやらかしのカテゴリ・影響範囲・関係性を元に0〜180°の角度スコアを算出する

**AC-2: 角度がピクトグラム+スタンプ+SE音でビジュアル表示される**
- **Given** 角度スコアが算出されたとき
- **When** 診断結果画面が表示される
- **Then** 下記の角度ステージに応じたピクトグラム画像（頭を下げる人物シルエット）がスタンプ演出（ドン！と打ち付け）＋SE音で表示され、ラベルが表示される

| 角度 | ステージ名 | 説明 |
|------|-----------|------|
| 0〜15° | 会釈 | まだ謝罪の入口 |
| 16〜45° | 深謝 | ちゃんと謝る領域 |
| 46〜90° | 土下座 | 床との交渉開始 |
| 91〜120° | 寝下座 | 反省が水平化 |
| 121〜150° | 焦げ下座 | 床の温度が上がる |
| 151〜180° | 焼き寝下座 | 謝罪が炭火焼きになる |

**AC-3: 角度の根拠が説明される**
- **Given** 角度スコアが表示されたとき
- **When** 根拠セクションを確認する
- **Then** 「影響範囲が広い」「期限超過あり」「過去の信頼残高が低い」等、角度を決定した要因がリスト表示される

## US-208: AI角度 vs 自己申告のギャップを分析する

| 項目 | 内容 |
|------|------|
| **ペルソナ** | Kenta / Seiichi |
| **優先度** | P0 |
| **Story Points** | 5 |
| **依存** | US-207 |
| **Epic** | Epic 2 |

**ユーザーとして**、自分が思う謝罪の深刻さとAIの判断のズレを知りたい。  
**なぜなら**、自己評価が甘い場合の危険性と、過剰謝罪のリスクを両方把握したいから。

### 受け入れ条件

**AC-1: 自己申告の角度を入力できる**
- **Given** AI角度アセスメント結果が表示されているとき
- **When** 「あなたが思う謝罪の深刻度は？」のスライダーまたは選択UIを操作する
- **Then** 0〜180°の範囲で自己申告角度が入力できる

**AC-2: ギャップ分析が表示される**
- **Given** AI角度と自己申告角度が揃ったとき
- **When** ギャップ分析セクションを確認する
- **Then** 2つの角度の差分と、差が示す意味（「AIより甘く見ている」「過剰に心配している」）が表示される

**AC-3: ギャップへのアドバイスが提示される**
- **Given** ギャップが15°以上ある場合
- **When** アドバイスセクションを確認する
- **Then** 「相手はあなたが思うより怒っている可能性があります。謝罪プランを慎重に作成してください」等の具体的なアドバイスが表示される

## INVEST基準チェック

| 基準 | 適合状況 |
|------|---------|
| **Independent** | 各ストーリーは依存を明示しつつ、個別にテスト・リリース可能 |
| **Negotiable** | 受け入れ条件は具体的だが、実装手法は柔軟に変更可能 |
| **Valuable** | 各ストーリーがユーザーに直接的な価値を提供する |
| **Estimable** | Story Points で相対見積もり済み（3〜13 SP） |
| **Small** | 1ストーリーが1〜3日で実装可能な粒度 |
| **Testable** | Given/When/Then 形式で受け入れ条件を記述 |
