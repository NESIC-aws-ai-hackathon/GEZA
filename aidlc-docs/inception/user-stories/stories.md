## プロダクトコンセプト

GEZAは**謝罪行動支援AI**です。  
「やらかした」と一言入れるだけで、謝罪角度判定→台本フル生成→タイミング・手土産提案まで全部AIがやります。あなたは頭を下げるだけ。

> **AI整え、人間が向き合う** — AIが分析・構成・演出を担い、人間は「誠意を伝える」一点に集中できる。

**3フェーズ支援（謝罪ライフサイクル）**:
- **謝罪前**: 角度アセスメント → 台本生成 → 準備サポート（予選MVP）
- **謝罪中**: 怒り残量スキャナー → GEZA耳打ちモード（決勝拡張: Journey 10）
- **謝罪後**: 再発防止策 → フォローメール → カルテ蓄積

```
コア機能（入力一つで全自動）
  やらかし深掘り分析    ← AIが追加質問を繰り返し、本人すら気づいていない本質を炙り出す
  謝罪角度アセスメント  ← 深掘り結果を元にAIが0〜180°で判定・6ゾーン14段階演出
  謝罪台本フル生成      ← 相手分析 → NGワード → 第一声 → 全セリフ台本
  タイミング・手土産    ← いつ・どこで・何を持っていくかまで指示
  準備サポート          ← チェックリスト・フォローメール・再発防止策

オプション（やりたい人だけ）
  リハーサルモード      ← AI製台本の読み合わせ。失敗してもAIが書き直す
  上司向け指導練習      ← 独立フロー
```

**キャッチコピー：「土下座はあなたに、誠意はAIに。」**

## 感情定義（200種類 / 15カテゴリ）

ストーリー内で参照される謝罪ボスの感情ラベル一覧。facesjs SVG + CSS表情制御で表現する。  
AIは **カテゴリ** を選択し、同一カテゴリ内の感情をランダムに遷移させることで、人間らしい自然な表情揺らぎを実現する。

### 設計方針

| 項目 | 内容 |
|------|------|
| 感情総数 | **200種類**（15カテゴリ × 平均13〜14感情） |
| AIの返却単位 | **カテゴリID**（例: `fierce_anger`） — 個別感情は FE がカテゴリ内でランダム選択 |
| 表情切り替え | カテゴリ内ランダム遷移: 2〜4秒間隔で同カテゴリの別感情に自然に変化（急激な切り替えを避ける補間付き） |
| アクション演出 | **大げさモーション**を基本とし、ビジュアルインパクトを最大化（画面揺れ・明暗変化・体揺れ・後ずさり・前のめり等）|
| CSS表情制御 | 眉・目・口のCSS transform パラメータ × 200パターンを `emotions.js` に定義。カテゴリ間遷移は200ms ease-in-out 補間 |

| ルール | 内容 |
|--------|------|
| 遷移間隔 | 2〜4秒のランダム間隔（人間のマイクロ表情に近い自然なリズム） |

| カテゴリ間遷移 | AIが新しいカテゴリIDを返却した場合、300ms かけて旧カテゴリ→新カテゴリの中間表情を経由して遷移 |

### 受け入れ条件

**AC-1: トップ画面にアバターが表示される**
- **Given** ユーザーがアプリのURLにアクセスしたとき
- **When** トップ画面が読み込まれる
- **Then** facesjs SVGアバター（謝罪ボス）が画面上部中央に表示され、アイドルアニメーション（瞬き・視線移動・頭の揺れ）が再生される

- **Given** トップ画面が表示されたとき
- **When** ユーザーが画面を見る
- **Then** 「土下座はあなたに、誠意はAIに。」がアバター付近に表示される

**AC-3: ステータスゲージが表示される**
- **Given** トップ画面にアバターが表示されたとき
- **When** ユーザーがアバターを確認する
- **Then** 怒り度・信頼度・難易度の3ゲージが視覚的に表示される

**AC-4: モード選択導線がある**
- **Given** トップ画面が表示されたとき
- **When** ユーザーがモード選択エリアを見る
- **Then** 「実案件モード」「ストーリーモード」「謝罪カルテ」「上司向けフィードバック」への導線ボタンが表示される

**AC-5: スマホ表示に対応する**
- **Given** ユーザーがスマホ（375px幅）またはPCからアクセスしたとき
- **When** 画面が表示される
- **Then** スマホ幅375pxでレイアウトが崩れず自然に表示される。PCではスマホ幅で中央表示される

## US-1004: Web会議中に画面上で助言を受けながら謝罪する

| 項目 | 内容 |
|------|------|
| **ペルソナ** | Kenta / Misaki |
| **優先度** | P3（決勝拡張） |
| **Story Points** | 8 |
| **依存** | US-1001, US-1002 |
| **Epic** | Epic 10 |

**ユーザーとして**、Web会議（Zoom/Teams/Google Meet）中に謝罪する場面で、相手の反応を見ながらGEZAから画面上で助言を受けたい。  
**なぜなら**、会議相手にAI助言を聞かせずに、静かに謝罪の言葉選びを支援してほしいから。

### 受け入れ条件

**AC-1: Web会議モードを起動できる**
- **Given** GEZA謝罪中支援画面を開いた状態で
- **When** 「Web会議モード」を選択する
- **Then** PC音声出力（相手の声）とマイク入力（自分の声）を入力ソースとして設定し、Web会議向けUIに切り替わる

**AC-2: 助言が音声ではなく画面上に表示される**
- **Given** Web会議モードが有効な状態で
- **When** 危険発言や怒り残量の変化が検知される
- **Then** 助言は音声ではなく、専用タブ/前面パネルに短文テキストで静かに表示される（会議相手に聞こえないようにするため）

**AC-3: Web会議の邪魔にならないUIで表示される**
- **Given** Web会議中にGEZAパネルが表示されている状態で
- **When** 助言が表示される
- **Then** 短文中心（15文字以内）のコンパクトなUIで、会議画面を大きく遮らない表示となる

## US-1005: Web会議中に言い訳しそうになったら画面で警告してもらう

**ユーザーとして**、Web会議中に言い訳をしそうになったとき、GEZAに短い警告を画面表示してほしい。  
**なぜなら**、余計な一言で状況を悪化させることを防ぎたいから。

### 受け入れ条件

**AC-1: Web会議中のユーザー発話から危険発言を検知する**
- **Given** Web会議モードが有効で、マイク入力が取得されている状態で
- **When** ユーザーが言い訳・逆ギレ・責任転嫁・NGワードを発話しようとする

**AC-2: 短い警告テキストが表示される**
- **Given** 危険発言が検知された状態で
- **When** 警告パネルに通知される
- **Then** 「責任転嫁！」「言い訳注意」「まず謝意を」等の短い警告が表示され、3秒後に自動消去される
