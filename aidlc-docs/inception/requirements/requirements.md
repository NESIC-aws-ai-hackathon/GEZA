# GEZA 要件定義書

## インテント分析

| 項目 | 内容 |
|------|------|
| ユーザーリクエスト | 謝罪・クレーム対応・注意指導などの高リスク会話を、AIが生成した仮想相手に対して事前練習できるアプリ |
| リクエストタイプ | 新規プロジェクト（Greenfield） |
| スコープ | システム全体（Epic 1〜7、全ユーザーストーリー） |

## 技術スタック決定

| レイヤー | 技術 | 根拠 |
|----------|------|------|
| フロントエンド | HTML/CSS/Vanilla JS（SPA） | シンプルで素早い構築、依存関係なし |
| バックエンド | AWS Lambda（Python 3.12, 512MB, 30s） + API Gateway HTTP API v2 | サーバーレス、AGENTS.md準拠、プロトタイプ実証済 |
| LLM | AWS Bedrock（Amazon Nova Lite：軽量用途 / Claude Sonnet：高品質用途） | プロトタイプでNova Liteの1～3秒応答を実証 |
| 音声入力 | AWS Transcribe Streaming | リアルタイム音声→テキスト変換（Epic 4から実装） |
| 音声出力 | Amazon Polly Kazuha（ja-JP, Neural） + SpeechMarks（Viseme） | プロトタイプで口パク同期実証済 |
| アバター | facesjs v5.0.3（SVGアバター） + CSS表情制御（30感情） | MP4動画より辺度・速度・コスト全て優位、プロトタイプ実証済 |
| 認証 | AWS Cognito（User Pools） | 安全なユーザー管理、Epic 1から実装 |
| データベース | Amazon DynamoDB | サーバーレス、柔軟なスキーマ |
| 静的ホスティング | S3 + CloudFront | 高速グローバル配信 |

## AWS BedrockモデルのLLM用途別選定方針

| 用途 | 推奨モデル | 理由 |
|------|-----------|------|
| 謝罪相手の性格・怒りポイント・NGワード生成 | Claude Sonnet | 高品質なキャラクター生成が必要 |
| 謝罪内容の評価・追撃質問生成・感情ラベル返却 | Amazon Nova Lite | 1〜3秒応答、プロトタイプ実証済み |
| NGワード検知・感情分類 | Amazon Nova Lite | 高速レスポンスが必要 |
| フォローメール・再発防止策生成 | Amazon Nova Lite または Claude Sonnet | コスト効率と品質のバランス |
| ストーリー自動生成 | Claude Sonnet | 創造的なシナリオ生成が必要 |

## LLMプロンプト管理方針

- テンプレートは `backend/prompts/` に配置
- 変数は `{{variable_name}}` 形式
- プロンプト変更時は `docs/prompt-spec.md` を先に更新

## 制約・前提条件

- `.env` はgitにコミットしない
- フロントエンドからLLM APIを直接呼ばない
- 実在の人物名・企業名を謝罪ボスとして生成しない
- 法的助言・医療助言を生成しない
- MVPスコープ外の機能は実装しない
