# GEZA 要件定義書

## インテント分析

| 項目 | 内容 |
|------|------|
| ユーザーリクエスト | やらかし内容を一言入力するだけで、謝罪角度判定→台本フル生成→タイミング・手土産提案まで全部AIが自動実行する謝罪行動支援AI。謝罪前・謝罪中・謝罪後を一貫支援する謝罪ライフサイクル型アプリ |
| リクエストタイプ | 新規プロジェクト（Greenfield） |
| スコープ | システム全体（Epic 1〜10、41ユーザーストーリー / 271 SP） |
| 複雑度 | 複雑（マルチモーダル入力・動的UI・複数AWSサービス連携） |

> **スコープについて**: 本ハッカソンはAI-DLCの有効性を検証する場であるため、意図的に挑戦的なスコープを設定している。AI-DLCによる「高密度MVP」構築と「AI駆動開発の生産性検証」を示す目的で、広めのスコープとしている。予選MVPは U0+U1+U2、Epic 10（謝罪中支援）は P3 の決勝拡張構想であり、予選必達ではない。

> **コンセプト**: 「AIが考え、人間が詫びる」Human-in-the-Loop を謝罪の本番中にまで拡張し、謝罪ライフサイクル（Before/During/After）の完全カバーを実現する。

| FR-903 | 相手の発言から許容余地（許す余地の残量）を推定・表示する |

## AWS BedrockモデルのLLM用途別選定方針

> モデルは fast / standard / premium の3プロファイル制で管理。詳細は `application-design.md §1.2` 参照。

| 用途 | 推奨モデル | 理由 |
|------|-----------|------|
| 謝罪角度アセスメント・謝罪評価・感情分類・NGワード検知・カルテ分析 | Amazon Nova Lite（fast） | 1～3秒応答、プロトタイプ実証済み |
| 謝罪プラン生成・深掘り分析 | Claude Haiku 4.5（standard） | コストコンシェルかつ十分な品質 |
| 謝罪相手生成・ストーリー生成 | Claude Sonnet（premium） | 高品質なキャラクター・シナリオ生成が必要 |
| フィードバック・再発防止策・フォローメール・返信分析生成 | Claude Sonnet（premium） | 構造化された高品質文章が必要 |
| 指導評価 | Amazon Nova Lite（fast） | 高速レスポンスが必要 |
| 指導フィードバック生成 | Claude Sonnet（premium） | 建設的な改善スクリプト生成 |
| 怒り残量スキャナー（謝罪中支援） | Amazon Nova Lite（fast） | リアルタイム感情分析に高速応答が必須 |
| GEZA耳打ちモード（謝罪中支援） | Amazon Nova Lite（fast） | 危険発言検知に低レイテンシが必須 |
